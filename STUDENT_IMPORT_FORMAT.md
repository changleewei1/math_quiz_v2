# 學生帳號批次匯入格式說明

## 📋 概述

學生帳號可以透過 JSON 格式批次匯入，每個學生需要提供姓名和密碼。

## 📝 JSON 格式

輸入必須是一個 JSON **陣列**，每個元素代表一名學生。

### 基本結構

```json
[
  {
    "name": "學生姓名",
    "password": "密碼"
  },
  {
    "name": "另一個學生",
    "password": "另一個密碼"
  }
]
```

## 📋 欄位說明

### 必要欄位

| 欄位名稱 | 類型 | 說明 | 範例 |
|---------|------|------|------|
| `name` | string | 學生姓名（必須唯一） | `"王小明"` |
| `password` | string | 學生密碼（會自動加密儲存） | `"abc123"` |

### 可選欄位

| 欄位名稱 | 類型 | 說明 | 預設值 | 範例 |
|---------|------|------|--------|------|
| `is_active` | boolean | 是否啟用帳號 | `true` | `true` |

## ✅ 驗證規則

系統會自動驗證以下項目：

1. ✅ JSON 格式是否正確
2. ✅ 資料是否為陣列
3. ✅ 每個學生是否包含 `name` 和 `password`
4. ✅ `name` 是否為非空字串
5. ✅ `password` 是否為非空字串
6. ✅ 學生姓名是否重複（資料庫中已存在）

## ❌ 常見錯誤

### 錯誤 1：JSON 格式錯誤

**錯誤範例：**
```json
{
  "name": "王小明",
  "password": "abc123"
  // 缺少逗號或括號
}
```

**正確：**
```json
[
  {
    "name": "王小明",
    "password": "abc123"
  }
]
```

### 錯誤 2：缺少必要欄位

**錯誤範例：**
```json
[
  {
    "name": "王小明"
    // 缺少 password
  }
]
```

**正確：**
```json
[
  {
    "name": "王小明",
    "password": "Wang2024!"
  }
]
```

### 錯誤 3：學生姓名重複

**錯誤：**
如果資料庫中已有「王小明」，再次匯入會失敗。

**解決方法：**
- 檢查是否已存在該學生
- 或使用不同的姓名
- 或先刪除舊的學生資料

## 💡 密碼建議

### 安全密碼格式

建議使用以下格式生成密碼：

1. **姓名拼音 + 年份 + 特殊符號**
   - 王小明 → `Wang2024!`
   - 李小華 → `Li2024@`

2. **姓名拼音 + 數字 + 特殊符號**
   - 張小美 → `Zhang123#`
   - 陳小強 → `Chen456$`

3. **隨機密碼（推薦）**
   - 使用 ChatGPT 生成隨機密碼
   - 例如：`K8m#P2n$Q9w`

### 密碼強度要求

- ✅ 長度：至少 8 個字元
- ✅ 包含：大寫字母、小寫字母、數字、特殊符號
- ✅ 避免：常見密碼（如：123456、password）

## 📤 使用方式

1. 在後台管理頁面，點擊「學生管理」
2. 點擊「批次匯入學生」
3. 將 JSON 內容貼上到輸入框
4. 點擊「批次匯入」完成匯入

## 🔒 安全性說明

- 密碼會使用 SHA-256 加密後儲存
- 原始密碼不會被保存
- 登入時會驗證加密後的密碼

## 📝 提示

1. **使用 ChatGPT 生成密碼**：明確要求生成安全密碼
2. **保留原始 JSON 檔案**：以便日後修改或重新匯入
3. **先測試少量學生**：確認格式正確後再大量匯入
4. **檢查姓名是否重複**：避免匯入失敗
